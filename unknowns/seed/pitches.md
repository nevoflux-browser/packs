# Pitches

**Stage:** Communication | **Mode:** Structured Generation | **Targets:** Known unknowns

## What it does

Packages your work into a self-contained deliverable document for a specific audience to get
buy-in. Supports comparing pitches across 2-3 different audiences.

## When to use it

- When you need approval or buy-in from others
- When you need to explain your work to people who weren't part of the process
- When communicating to multiple audiences with different concerns
- When you want to test if your framing holds up across different viewpoints

## How it works

1. You specify what to pitch and to whom (1-3 audiences)
2. The agent gathers outputs from any previous unknowns techniques used in the conversation
3. For each audience, packages everything into a cohesive document framed for their concerns
4. If multiple audiences, adds cross-audience comparison

## Key mechanism

**Packaging, not analysis.** This is not another critical thinking tool. It's a communication
tool. The hard work of thinking has already been done (via other techniques). This step is about
presenting that work in the most effective way for each audience.

**Audience-specific framing.** The same work needs different framing for different people. A
technical reviewer cares about tradeoffs considered. A business stakeholder cares about impact
and timeline. A team member cares about what changes for them. The pitch restructures the same
content around each audience's concerns.

**Cross-audience comparison.** When you pitch to multiple audiences, comparing the versions
reveals where your messaging is inconsistent. If you frame something one way for engineers and
a contradictory way for executives, that's a signal you haven't fully resolved the underlying
tension.

## Output

Self-contained pitch document(s) per audience — something you can send directly without
additional context. Plus cross-audience notes if multiple audiences.

## Connects to

- After this → `quizzes` if cross-audience gaps suggest you don't fully understand a topic
- Before this ← `notes` (package documented execution)
- Before this ← `plans` (pitch a proposed plan for approval)
- Before this ← `brainstorming` + `blind-spot-pass` (pitch a well-examined idea)
